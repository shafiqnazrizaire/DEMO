# Project Audit – Kemboja Juara Travels (travel-website)

**Audit date:** 2026-02-09  
**Scope:** Entry points, environment variables, logic, conflicts, and verification steps.

---

## 1. Project structure (relevant files)

| Path | Purpose |
|------|--------|
| **server-stripe.js** | **Primary app server** – Stripe Checkout, webhooks, Supabase, emails, static files. Uses **PORT 3000** (from .env). |
| **server.js** | Legacy/email-only server – `/api/book`, `/api/test-email`, `/api/health`. Uses PORT from .env (default 8080 in code). |
| **index.html** | Main site + booking form. Calls `/api/create-checkout-session` (relative URL). |
| **success.html** | Post-payment success page. |
| **cancel.html** | Payment cancelled page. |
| **script.js** | Front-end logic, form, Stripe redirect. |
| **.env** | Loaded by `require('dotenv').config()` in server-stripe.js. **Not in .gitignore** – see conflicts. |
| **scripts/generate-favicons.js** | Generates favicon-16/32/48/180 from `Kmbja logo.png`. |

## 2. Entry points

| Entry | Command | Port | Serves |
|-------|--------|------|--------|
| **Intended for full app** | `node server-stripe.js` | **3000** | Static (index, success, cancel), `/api/create-checkout-session`, `/api/webhook`, `/api/health`, `/api/db-test`, Supabase schema on startup. |
| **package.json "start"** | `npm start` → `node server.js` | 3000 if PORT in .env, else 8080 | Email APIs only; **no Stripe, no Supabase, no checkout**. |

**Conflict:** `npm start` runs **server.js**, but booking, Stripe, and Supabase run on **server-stripe.js**. Use the commands in Section 6 to run and verify the full app.

|--------|------|--------|

| POST | /api/create-checkout-session | Create Stripe Checkout session; redirects to Stripe. |

| GET | /api/db-test | Supabase query test; returns DB name and time. |
| Static | /* | index.html, success.html, cancel.html, CSS, JS, images. |

---

## 5. Recent logic (server-stripe.js)

- **Database:** Only **DATABASE_URL** (trimmed); Supabase pool with SSL; `pool` null if URL missing.
- **Startup:** Tests DB with `SELECT 1`; on success runs `createDatabaseSchema()` (CREATE TABLE IF NOT EXISTS + indexes).
- **Bookings:** Saved in `saveBookingToDatabase(session)`; full description in metadata (description + description_2); emails (customer + company) include full description.
- **Webhook:** `handleStripeWebhook` verifies signature with **STRIPE_WEBHOOK_SECRET**; on `checkout.session.completed` saves booking and sends both emails.
- **Favicon:** index/success/cancel use `Kmbja logo.png` + favicon-32x32.png etc.; files generated by `npm run favicons`.

## 6. File / config conflicts and risks

| Issue | Severity | Action |
|-------|----------|--------|
| **.env not in .gitignore** | High | Add `.env` to `.gitignore` so secrets are not committed. |
| **package.json "start" runs server.js** | Medium | For full app, run `node server-stripe.js` (or add a `start:stripe` script). |
| **test-email / health scripts use port 8080** | Low | These target server.js. For server-stripe.js use port 3000 (e.g. `http://localhost:3000/api/health`). |

## 7. Exact commands to restart and verify

### 7.1 Stop any process on port 3000 (PowerShell)

```powershell
Get-NetTCPConnection -LocalPort 3000 -ErrorAction SilentlyContinue | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force -ErrorAction SilentlyContinue }
```

### 7.2 Start the server (full app – Stripe + Supabase + emails)

```powershell
cd "c:\Users\Nazeem\OneDrive\Documents\Assignment 2026\KEMBOJA WEBSITE V1 NEW\travel-website"
node server-stripe.js
```

**Expected startup output:**

```
🚀 Server running on port 3000
💳 Stripe integration active
🔔 Webhook endpoint: /api/webhook
🗄️ Supabase database connected
✅ Database schema created/verified
```

If **DATABASE_URL** is missing or wrong you will see:

- `❌ Database connection failed: <message>`  
- `⚠️ DATABASE_URL not set in .env – database disabled`

### 7.3 Verify features (run with server already running)

1. **Health + DB**
   - Browser or PowerShell:  
     `Invoke-RestMethod -Uri http://localhost:3000/api/health`
   - Expect: `status: 'ok', database: 'connected'`

2. **DB test**
   - `Invoke-RestMethod -Uri http://localhost:3000/api/db-test`
   - Expect: `ok: true`, `message: 'Supabase database connection is working'`

3. **Static site**
   - Open: http://localhost:3000  
   - Expect: index.html with favicon and booking form.

4. **Checkout (end-to-end)**
   - Submit a test booking from the form; use Stripe test card `4242 4242 4242 4242`.
   - Expect: Redirect to Stripe → success/cancel → customer email + company email + row in Supabase `bookings` table.

5. **Webhook (if using Stripe CLI)**
   - `stripe listen --forward-to localhost:3000/api/webhook`
   - Use the CLI’s webhook signing secret in .env as **STRIPE_WEBHOOK_SECRET** for local testing.

- **Single command to run the full app:**  
  `node server-stripe.js`
